# Development Notes

This folder contains intermediate documentation files created during the development and debugging process. These files are useful for:

- **Debugging**: Tracking issues and their resolutions
- **Documentation**: Recording decisions and methodology changes
- **LLM Prompts**: Prompts used to seek expert guidance
- **Status Reports**: Progress summaries and next steps
- **Fix Summaries**: Documentation of fixes and corrections

## Organization

Files are kept here for reference but are not essential for running the pipeline. The main documentation is in the root directory:
- `README.md` - Main project README
- `START_HERE.md` - Quick start guide
- `CONTRIBUTING.md` - Contribution guidelines

## Key Files

- **Expert Guidance**: Files starting with `EXPERT_GUIDANCE_*` contain guidance from domain experts
- **Prompts**: Files starting with `PROMPT_FOR_LLM_*` contain prompts used for seeking second opinions
- **Status Reports**: Files with `STATUS_*`, `NEXT_STEPS_*`, `FINAL_*` contain progress summaries
- **Fix Documentation**: Files with `FIX_*`, `VALIDATION_*`, `CRITICAL_*` document fixes and corrections
