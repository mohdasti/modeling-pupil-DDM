# Intermediary Scripts

This folder contains utility scripts, test scripts, and intermediate analysis scripts that were created during development and debugging.

## Script Categories

### Test Scripts (`test_*.R`)
- Scripts for testing specific functionality or debugging issues
- Unit tests for data processing steps
- Validation scripts

### Utility Scripts (`check_*.R`, `verify_*.R`, `audit_*.R`, `update_*.R`)
- Data validation and quality checks
- Model verification scripts
- Path update utilities
- Audit scripts

### Pipeline Scripts (`run_*.R`, `run_*.sh`)
- Various pipeline execution scripts created during development
- Background execution scripts
- Monitoring scripts

### Fix Scripts (`fix_*.R`)
- Scripts created to fix specific issues
- One-time correction scripts

## Usage

These scripts are kept for reference and debugging purposes. For running the main analysis pipeline, see:
- Main pipeline scripts in the `scripts/` directory
- `START_HERE.md` in the root directory

## Note

When the project reaches a stable state, these intermediary scripts can be reviewed and potentially archived or removed, keeping only the essential pipeline scripts.
